ШАГ 1. Определим максимальное число клиентов, которых можно обслужить
	   Для этого переберем все возможные комбинации номиналов в запросах клиентов,
	   и выберем те комбинации, при которых будет обслужено максимальное число клиентов.

Результат:
1.1) Максимальное число обслуженных клиентов: 3
1.2) Возможные комбинации удовлетворения запросов:
	a = [0, 0, 0, 0]
	a = [1, 0, 0, 0]
	a = [1, 1, 0, 0]
	a = [1, 1, 1, 0]
, где i-й элемент соответствует a[i] запросу i-го клиента,
а "0" означает, что данный клиент не рассматривается
(вся нумерация начинается с единицы)

ШАГ 2. Определим минимальное число отказов среди всех запросов.
	   Для этого пройдемся по всем найденным комбинациям запросов и для каждой
	   из них будем удовлетворять все оставшиеся запросы, пока позволяет фонд.
       Таким образом мы найдем минимальное число отказов для данной комбинации,
       после чего нам останется лишь выбрать наиболее оптимальную комбинацию.
Результат:
2.1) Минимальное число отказов: 1
2.2) Комбинации, удовлетворяющие этому условию:
	a = [0, 0, 0, 0]
	a = [1, 0, 0, 0]
	a = [1, 1, 0, 0]
	a = [1, 1, 1, 0]
(если комбинаций несоклько, программой будет выбрана первая)

ШАГ 3. Теперь, когда найдена итоговая комбинация удовлетворения запросов,
